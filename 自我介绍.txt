	Lambda： 支持
		概念1：
		是AWS在2014年推出的「无服务器」（Serverless）计算服务，用户无需管理服务器，可以更专注自己业务。由于上手简单，而且真正利用了云的优势
		「无服务器」并不是真的没有服务器，而是由云计算厂商将管理服务器的繁杂事务自动化，为用户提供更贴近业务的接口
		在 Lambda 里，使用函数（Function）来存储代码，供 Lambda 调用。也正因此，Lambda类的无服务器计算服务也被称作「函数即服务」（Function-as-a-Service / FaaS）。
		https://zhuanlan.zhihu.com/p/106513886
		
		概念2：
		Lambda 是 AWS 提供的一种计算服务，无需管理或配置服务器，因为 lambda 是serverless。可以根据你的代码进行缩放，通俗点说就是当程序需要处理大量task时，lambda会同时调用很多个lambda，可以达到500-3000个。所以处理起来就会很快。
		https://blog.csdn.net/libao_sir/article/details/111224830
		
		Lambda 为 .NET (PowerShell,C#)、Go、Java、Node.js、Python 和 Ruby 提供运行时。
		
		超过 50MB 的部署程序包 -> 如果部署程序包大于 50 MB，建议将函数代码和依赖项上载到 Amazon S3 存储桶。
		
		对于编译的语言，您必须创建 .zip 归档部署包才能上传 Lambda 函数代码。
		对于非编译的语言，Lambda 控制台为非编译语言提供了代码编辑器，使您可以快速修改和测试代码
		
		https://docs.aws.amazon.com/zh_cn/lambda/latest/dg/getting-started.html

		CloudFront 提供最先进的安全功能，包括领域级加密和 HTTPS 支持，与AWS Shield、AWS Web 应用防火墙和 Amazon Route 53 无缝集成，以防止多种类型的攻击，包括网络和应用层 DDoS 攻击
		CloudFront 可以无缝使用任何 AWS 源，例如 Amazon S3、Amazon EC2, Elastic Load Balancing 或任何自定义 HTTP 源
